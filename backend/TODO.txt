 Here's how you could approach building this:


  Step 1: Create the Files and Types
   * Create src/layers/dataLinkLayer_2.ts.
   * In src/types/index.ts, define a DataLinkLayerData interface. It should include properties like sourceMac, destinationMac, and fcs (for the checksum
     value).


  Step 2: Update the `Orchestrator`
   * This is a key architectural change. You need to insert your new DataLinkLayer into each of your device stacks, right between the NetworkLayer and the
     PhysicalLayer.
   * This is also where you should define the hardcoded MAC addresses for each device.


  Step 3: The "ARP Table" Simulation
  How does Host A's Data Link Layer know the MAC address of the router to send the frame to? In the real world, it uses ARP. We can simulate this easily.
   * Hint: In your Orchestrator, create a simple Map that connects IP addresses to the MAC addresses you defined. new Map([ ["192.168.1.10",
     "AA:AA:AA:AA:AA:AA"], ... ]).
   * You can then pass this "ARP table" into the constructor of your DataLinkLayer instances.


  Step 4: Implement `handleOutgoing` in `DataLinkLayer`
   * This method will receive a packet from the NetworkLayer.
   * It will look at the packet's ultimate destinationIp to figure out the next hop's IP (for now, always the router).
   * It will use its "ARP table" to find the MAC address for that next hop IP.
   * It will calculate the FCS (your simple checksum).
   * It will add the new DataLinkLayer header and trailer.
   * Finally, it will pass the completed frame down to the PhysicalLayer.
